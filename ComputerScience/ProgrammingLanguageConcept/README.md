## 개요

>### 프로그래밍 언어란
>- 계산 과정을 기계가 읽을 수 있고 사람이 읽을 수 있도록 기술하기 위한 일종의 표기법
>- 주요 언어 이론
>>- 구문법
>>>- 문장을 구성하는 방법
>>- 의미론
>>>- 작성된 문장의 의미를 결정
>>- 타입 시스템
>>>- 타입의 올바른 사용법

>### 프로그래밍 언어의 종류
>- 프로그래밍 패러다임
>> 프로그래밍을 하는 기본적인 스타일
>>- 명령형 프로그래밍 (imperative programming)
>>>- 문제를 해결하는 절차를 기술 혹은 명령하는 방식의 프로그래밍 스타일
>>>- 수행할 명령어들로 구성되며 명령어들은 주로 프로그램의 상태를 변경하는 역할을 함
>>- 함수형 프로그래밍 (functional programming)
>>>- 프로그램의 계산 과정을 수학 함수의 실행으로 간주하는 프로그래밍 스타일
>>>- 함수들로 구성되며 순수한 함수는 원칙적으로 부수 효과(상태 변경 혹은 데이터 수정)을 허용하지 않으나 함수형 언어는 언어에 따라 부수 효과를 허용함
>>- 논리 프로그래밍 (login programming)
>>>- 술어 논리(predicate login)을 기반으로 한 프로그래밍 스타일
>>>- 프로그램을 문제에 대한 사실 혹은 규칙을 표현하는 논리 문장들의 집합으로 구성함
>>- 객체지향 프로그래밍 (object-oriented programming)
>>>- 실세계를 모의실험하기 위한 언어로 고안되었으며 객체 개념을 기반으로 하는 프로그래밍 스타일
>>>- 객체는 속성과 그 속성에 대한 행동(함수, 메서드)들을 포괄하는 개념임
>>>- 프로그램 실행은 객체 사이의 상호작용에 의해 이루어짐
>- 프로그래밍 언어의 종류
>>- 명령형 언어
>>>- C - 벨 연구소에서 유닉스 시스템 프로그래밍을 위한 언어로 개발되었으며, 대표적인 명령형 언어
>>>- Pascal - 교육용 프로그래밍 언어로 작고, 단순하고, 효율적이고, 구조화된 언어로 개발, 대표적 블록 구조 언어
>>>- Ada -1980년대 미국 국방성에서 개발한 명령형 블록 구조 언어로 추상 자료형, 병행성, 예외 처리 등의 최신 기능을 포함
>>>- Python - 현재 널리 사용되는 범용 언어로 코드 가독성을 강조하여 간단명료하게 설계된 인터프리터 방식의 대화형 언어로서 명령형, 객체지향 등의 여러 프로그래밍 패러다임을 지원함
>>- 객체지향 언어
>>>- C++ - C언어에 객체 기능을 추가
>>>- 범용 객체지향 언어로써 웹 및 모바일 응용 프로그램 개발에 가장 많이 사용되고 있고, 한 번 작성하면 어디서나 실행될 수 있도록 설계됨
>>>- Objective-C - 애플에서 개발한 범용 객체지향 언어로 C언어에 Smailltalk 스타일의 메시지 전달을 추가하여 개발한 언어
>>>- C# - Java에 영향을 받아 MS에서 개발한 닷넷 기반의 범용 객체지향언어
>>>- Visual Basic - Basic 에서 발전한 객체지향 언어, 초보자용 언어로 개발됨
>>- 함수형 언어
>>>- Lisp(LISt Processor), Scheme - MIT에서 리스트 처리 언어로 개발, Scheme는 LISP를 개선하여 개발된 언어, 수학의 함수를 기반으로 하는 함수형 언어
>>>- ML - 안전한 타입 시스템을 갖춘 범용 함수형 언어로 이후 OCaml 형태로 발전
>>>- Haskell - 순수 함수형 프로그래밍 언어를 목표로 개발, I/O와 같이 필요한 경우가 아니면 부수 효과가 없는 순수 함수로만 만들어짐
>>- 논리 언어
>>>- Prolog - 문제의 해결 절차가 아니라 해결해야 할 문제가 무엇인지 선언적으로 프로그래밍 하는 것이 가능
>>- 기타 언어
>>>- SQL - 데이터베이스 질의 언어
>>>- Perl, PHP - 웹 서버 프로그램 작성을 위한 스크립트 언어
>>>- Shell - 유닉스/리눅스 에서 사용되는 쉘 스크립트 언어
>- 함수형 프로그래밍 언어
>>- 변수나 배정문이 없으며 대신 함수의 매개변수를 통해서 값을 전달
>>- 함수형에서 반복은 재귀호출을 통해서 할 수 있음
>>- 장점
>>>- 프로그래밍 모델이 기계 모델과는 무관
>>>- 수학을 기반으로 하고 있으므로 프로그램의 의미를 명확하게 파악할 수 있음
>- 논리 프로그래밍 언어
>>- 문제에 대한 사실 혹은 규칙을 표현하는 논리 문장들의 집합으로 프로그래밍
>>- 문제를 해결하는 방법보다 문제가 무엇인지를 문제에 대한 사실 혹은 규칙 형태로 표현하는 논리 문장들로 구성됨
>- 객체지향 프로그래밍 언어
>>- 각 객체들은 스스로가 자신만의 기억 장치와 연산을 가지고 있는 컴퓨터처럼 기능
>>- 동일한 성질을 가진 객체들을 표현하는 클래스를 정의할 수 있으며 클래스는 객체에 대한 설계도 혹은 타입 역할을 함
>>- 프로그램 내에서 클래스의 특정한 사례인 실체(instance)로서의 객체를 생성하며, 이 객체들 사이의 상호작용(주로 메서드 호출)에 의해 프로그램이 실행됨

>### 프로그래밍 언어의 역사
>- 1950년대: 고급 프로그래밍 언어의 시작
>>- FORTRAN
>>>- 최초의 고급 프로그래밍 언어였기 때문에 대부분의 기능들은 새로운 기능이었고 일부는 이후의 언어에서 표준이 됨
>>>- 배열, 첨자 변수(index variable)에 의해서 제어되는 for 반복문, 분기 if문 등이 있음
>>- COBOL
>>>- 지금도 메인프레임 컴퓨터의 레거시 응용프로그램들에 사용되고 있으며 대용량 일괄 처리 및 트랜잭션 처리와 같은 작업에 많이 쓰임
>>>- 복잡한 알고리즘을 프로그래밍하기에 대단히 어려움
>>>- 데이터를 조직하는 레코드 구조와 프로그램의 실행부분과 분리된 자료구조, 다양한 출력 기능 등이 있음
>>- LISP
>>>- 리스트 자료구조와 함수 적용을 기반으로한 인공지능용 언어로 개발되었고 재귀호출이 매우 일반적으로 많이 사용됨
>>>- 실행 시간에 기억장소를 관리하는 매커니즘으로 더 이상 사용하지 않는 기억장소를 자동 반환하는 쓰레기 수집(garbage collection) 기능을 선구적으로 도입
>>>- 최근의 번역 기술의 발전과 기계 성능의 향상으로 함수형 언어가 일반적 프로그래밍을 위해 더욱 유용하게 사용될 수 있게 됨에 따라 영향력이 갈수록 증가되어 최근에도 많이 사용되고 있음
>- 1960년대: 프로그래밍 언어의 다양성
>>- Algol60.68
>>>- 알고리즘을 기술하기 위한 범용 언어를 목표로 개발
>>>- 현대의 대부분 명령형 언어들은 Algol의 영향을 받아 개발됨
>>>- 오늘날 알고리즘을 기술하기 위하여 Algol 형태의 구문이 많이 사용됨
>>>- 구조적 문장, begin-end 블록, 자유 양식, 변수의 타입 선언, 재귀 호출, 값 전달 매개변수 등 지금은 일반화된 많은 새로운 개념을 도입
>>>- 블록-구조 언어로 스택-기반 실행환경을 자연스럽게 도입하여 오늘날의 언어들에서도 중요한 구현방법으로 사용됨
>>>- 구문을 정의하는 방법으로 Backus-Naur 형식(BNF)을 최초로 사용
>>- PL/I
>>>- 병행성, 예외 처리 같은 시대를 앞선 기능을 많이 포함하였지만 충분히 이해되고 활용되지 못함
>>- Simula-67
>>>- 객체와 클래스 개념을 소개함으로써 추상화에 근본적으로 공헌하였고 객체지향 언어의 원조가 됨
>>- BASIC
>>>- 단순성 때문에 교육용 언어와 개인용 컴퓨터 응용을 위한 언어로 널리 사용됨
>- 1970년대: 단순성 및 새로운 언어의 추구
>>- PASCAL
>>>- 교육용 언어로 매우 성공적
>>- C
>>>- 실질적으로 모든 컴퓨터 시스템에서 사용할 수 있도록 설계된 프로그래밍 언어로써 생성된 프로그램이 매우 높은 성능을 가짐
>>>- 유닉스 운영체제의 성공과 보급에 따라 매우 대중적으로 사용되었고 다른 많은 운영체제에서도 채택되어 사용함
>>>- 운영체제 개발을 위한 언어로 개발되었기 때문에 작동하는 기계에 대해 많은 접근을 제공하여 중급 언어라고도 불림
>- 1980년대: 추상 자료형과 객체지향
>>- Ada
>>>- 추상 자료형인 패키지, 병행 혹은 병렬 프로그래밍 기능인 태스크, 예외 처리 등과 같은 기능을 포함
>>- Smalltalk
>>>- 객체지향 언어 중 가장 순수한 객체 지향 언어로 인정받고 있음
>>>- 최초로 GUI를 제공하는 언어였고, 매킨토시시의 그래픽 사용자 인터페이스에 영향을 줌
>>- C++
>>>- C언어의 구조체를 클래스 형태로 확장하여 C 언어의 효율성을 유지하면서도 객체지향 프로그래밍이 가능하게 함
>- 1990년대: 인터넷 언어와 새로운 시도
>>- Python
>>>- 플랫폼 독립성, 객체지향, 동적 타입 등의 특징을 갖는 언어
>>>- 문법이 매우 쉬워서 초보자들이 처음 프로그래밍을 배울때 많이 사용되는 언어
>>>- 학습용으로 좋은 언어인 동시에 실사용률과 생산성도 높은 강력한 언어로 접근성과 응용력이 좋음
>>>- 최근 빅데이터와 관련된 응용 분야에서 많이 사용됨
>>- Java
>>>- 선마이크로시스템즈의 제임스 고슬링이 개발한 객체지향 프로그래밍 언어
>>>- 가전제품에 탑재해 동작하는 프로그램을 위해 개발하였으나 현재 웹 애플리케이션 개발과 모바일 기기용 소프트웨어 개발에 가장 많이 사용하는 언어
>>>- 문법적인 특성은 C와 비슷하지만 컴파일된 코드가 플랫폼 독립적임
>>- JavaScript
>>>- 웹 브라우저 내에서 주로 사용하며 다른 응용 프로그램의 내장 객체에도 접근할 수 있음
>>>- Node.js 와 같은 런타임 환경과 같이 서버 사이드 네트워크 프로그래밍에도 사용됨
>- 2000년대: 새로운 미래를 향하여
>>- Scala
>>>- 객체지향 프로그래밍 언어와 함수형 프로그래밍의 요소가 결합된 다중패러다임 프로그래밍 언어
>>>- 자바 바이트코드를 사용하기 때문에 JVM에서 실행할 수 있고, 자바 언어와 호환되어 대부분의 자바 API를 그대로 사용할 수 있음
>>- Swift
>>>- iOS와 Mac OS를 위한 프로그래밍 언어
>>>- 클로저, 다중 리턴 타입, 네임스페이스, 제네릭, 타입 유추 등 Objective-C에 없었던 현대 프로그래밍 언어의 기능을 많이 포함
>>>- 코드 내부에서 C나 Objective-C 코드를 섞어서 프로그래밍하거나 스크립트 언어처럼 실시간으로 상호작용하며 프로그래밍할 수 있음

>### 추상화와 명령형 언어의 발전
>- 폰 노이만 모델 컴퓨터
>>- 실행할 프로그램을 메모리에 저장해서 이를 실행하는 방식으로, 프로그램 저장 방식 컴퓨터라고 함
>>- CPU는 주 메모리 내에 저장되어 있는 프로그램의 명령어를 한 번에 하나씩 읽어 들여 해석하고 실행하는데 이를 인출-해석-실행(fetch-decode-execute)주기라고 하고 CPU는 인출-해석-실행 주기를 계속해서 반복함
>- 추상화
>>- 실제적이고 구체적인 개념들을 요약하여 보다 높은 수준의 개념을 유도하는 과정
>>- 무엇을 요약하여 추상화하느냐에 따라 데이터 추상화와 명령어(제어) 추상화로 나뉨
>- 데이터 추상화
>>- 저 수준의 데이터나 자료형들을 요약하고 추상화하여 보다 고차원의 새로운 자료형 또는 자료구조를 만드는 과정을 의미
>>- 변수(variable)는 데이터 값을 저장하는 메모리 위치를 나타내는 추상화된 개념
>>- 자료형(data type)은 메모리에 저장될 값들의 종류에 대한 추상화된 개념
>- 제어 추상화
>>- 실행 흐름을 나타내는 저수준 명령어들을 요약하고 추상화하여 보다 높은 수준의 제어구조를 만드는 과정
>- 프로시저 추상화
>>- 프로시저나 함수도 제어 추상화를 통해 만들어진 기능으로 보다 복잡한 프로그램을 보다 쉽게 작성할 수 있음
>>- 크고 복잡한 프로그램을 쉽게 작성할 수 있는 여러 개의 프로시저로 구분하여 작성하게 함으로써 프로그래밍의 복잡도를 줄일 수 있음
>- 추상 자료형
>>- 데이터(자료구조)와 관련된 연산(프로시저)들을 한데 묶어 캡슐화하여 정의한 자료형
>>- 클래스가 추상 자료형의 발전된 형태라고 할 수 있으며 객체 지향 언어에서는 일반적으로 추상 자료형을 클래스, 인스턴스를 객체, 정의된 연산을 메서드(함수)라고 함

>### 프로그래밍 언어의 정의 및 구현
>- 프로그래밍 언어의 정의 방법
>>- 구문법(syntax)과 의미론(semantics)을 정의하는 것
>>>- 어휘구조
>>>>- 언어에서 사용하는 단어의 구조, 철자법등을 의미
>>>- 구문법
>>>>- 문장을 구성하는 방법을 의미
>>>- 의미론
>>>>- 작성된 문장 혹은 프로그램의 의미를 정하는 것
>- 프로그래밍 언어 구현
>>- 인터프리터
>>>- 소스 프로그램을 입력받아 구문법에 맞게 작성되었는지 검사하고 그 의미를 파악하여 의미에 맞게 동작하도록 해석
>>- 컴파일러
>>>- 소스 프로그램을 입력받아 구문법에 맞게 작성되었는지 검사하고 그 의미를 파악하여 의미에 맞게 동작하도록 기계어 명령어들로 번역하여 목적 프로그램을 생성
>>>- 생성된 목적 프로그램을 실행
