>### 네트워크의 구조
>- 네트워크
>>- 컴퓨터를 두 대 이상 연결하여 서로 데이터를 전송할 수 있는 통신망
>- 인터넷
>>- TCP/IP 프로토콜을 사용하여 전 세계 컴퓨터를 서로 연결하여 정보를 교환할 수 있도록 만든 하나의 거대한 통신망
>- 패킷
>>- 네트워크 통신을 할 때 사용되는 작게 분할된 데이터 조각으로 네트워크에서 전송하는 데이터의 기본 단위

<br>

>### LAN 과 WAN
>- LAN
>>- 집, 사무실, 학교 등의 건물과 같이 가까운 지역을 연결하는 네트워크
>- WAN
>>- 랜을 다시 하나로 묶는 거대한 네트워크
>>- 특정 도시, 국가, 대륙과 같이 넓은 범위에 구축되며 ISP(Internet Service Provider) 가 제공하는 서비스를 사용하여 구축된 네트워크
>>>- ISP
>>>>- 인터넷에 접속하는 수단을 제공하는 주체로써 일반 사용자, 기업체, 기관, 단체 등이 인터넷에 접속하여 인터넷을 사용할 수 있도록 돕는 사업자
>- &nbsp;|LAN|WAN
>   :--|:--|:--
>   범위|건물이나 특정 지역|랜과 랜을 연결
>   속도|빠르다|느리다
>   오류|적다|많다

<br>

>### 가정에서 하는 랜 구성
>- 인터넷 공유기 (broadband router) 를 중심으로 내부 인터넷 망을 구성하여 다양한 기기를 연결
>>- 최근, 라우터의 기능에 더하여 허브, 스위칭 허브, 방화벽 등 다양한 기능 추가 제공
>- 유선 랜과 무선 랜 방식으로 접속함

>### 회사에서 하는 랜 구성
>- DMZ (DeMilitarized Zone) 의 존재
>>- 외부 네트워크와 내부 네트워크 사이에 위치한 중간 지대(서브넷)
>>- 네트워크의 보안 영역으로 외부 공격자가 내부 네트워크에 침투하는 것을 막는 역할
>>- 외부에 공개하기 위한 네트워크
>>>- 웹 사이트를 불특정 다수의 외부 사용자에게 공개하기 위해 웹 서버 외부 공개
>>>- 외부 사용자와 메일을 주고받기 위한 메일 서버 외부 공개
>>>- 외부에서 도메인 이름을 사용하여 회사의 서버에 접속하기 위해 DNS 서버 외부 공개
>- 회사의 서버 설치 위치
>>- 온프레미스(on-premise)
>>>- 사내에 설치
>>>>- 서버 장비실에 랙(선반)에 서버를 두고 관리
>>>- 데이터 센터
>>- 클라우드

## 네트워크의 기본 규칙

>### OSI 모델과 TCP/IP 모델
>- 통신하기 위한 규칙을 프로토콜이라 함
>- OSI 모델
>>- ISO 국제표준화기구에서 정한 네트워크 표준규격
>>- 데이터의 송수신에서 일어나는 여러 가지 일을 7개 계층(레이어)으로 나눔
>>- 계층|이름|설명
>>   :--|:--|:--
>>   7계층|응용 계층(Application Layer)|이메일, 파일 전송, 웹 사이트 조회 등 애플리케이션에 대한 서비스를 제공
>>   6계층|표현 계층(Presentation Layer)|문자 코드, 압축, 암호화 등의 데이터를 변환
>>   5계층|세션 계층(Session Layer)|세션 체결, 통신 방식을 결정
>>   4계층|전송 계층(Transport Layer)|신뢰할 수 있는 통신을 구현
>>   3계층|네트워크 계층(Network Layer)|다른 네트워크와 통신하기 위한 경로 설정 및 논리 주소 결정
>>   2계층|데이터 링크 계층(Data Link Layer)|네트워크 기기 간의 데이터 전송 및 물리 주소를 결정
>>   1계층|물리 계층(Physical Layer)|시스템 간의 물리적인 연결과 전기 신호를 변환 및 제어함
>- TCP/IP 모델
>>- 네트워크 계층을 4개 계층(레이어)으로 나눔
>>- 응용 계층
>>>- 응용 계층 + 표현 계층 + 세션 계층
>>- 전송 계층
>>>- 전송 계층
>>- 인터넷 계층
>>>- 네트워크 계층
>>- 네트워크 접속 계층
>>>- 데이터 링크 계층 + 물리 계층

<br>

>### 캡슐화와 역캡슐화
>- 헤더
>>- 저장되거나 전송되는 데이터의 맨 앞에 위치하는 추가적인 정보 데이터
>>- 데이터의 내용이나 성격을 식별 또는 제어하는 데 사용
>- 캡슐화
>>- 상위 계층의 통신 프로토콜 정보를 데이터에 추가하여 하위 계층으로 전송하는 기술
>- 역캡슐화
>>- 상위 계층의 프로토콜에서 하위 계층에서 추가한 정보와 데이터를 분리하는 기술
>- 캡슐화와 역캡슐화의 흐름
>>- 계층|데이터 송신 측|데이터 수신 측
>>  :--|--:|:--
>>  응용 계층(세션계층+표현계층)|데이터|데이터
>>  전송 계층|1헤더+ 데이터|1헤더데이터 -1헤더
>>  네트워크 계층|2헤더+ 1헤더데이터|2헤더1헤더데이터 -2헤더
>>  데이터 링크 계층|3헤더+ 2헤더1헤더데이터 + 트레일러|3헤더2헤더1헤더데이터 -3헤더트레일러
>>  물리 계층|3헤더2헤더1헤더데이터트레일러|전기신호로 전송

## 물리 계층: 데이터를 전기 신호로 변환

>### 물리 계층의 역할과 랜 카드의 구조
>- 전기신호
>>- 물리계층은 0과 1의 비트열을 전기 신호로 변환
>>- 전기 신호의 종류
>>>- 아날로그 신호 - 물결 모양의 전기 신호
>>>- 디지털 신호 - 막대 모양의 전기 신호
>- 랜 카드
>>- 네트워크를 통해 데이터를 송수신할 때 0과 1의 정보를 전기 신호로 변환
>>- 컴퓨터의 네트워크 연결 및 데이터 전송을 담당하며 네트워크 카드 또는 네트워크 인터페이스 컨트롤러(NIC)라고도 불림

>### 케이블의 종류와 구조
>- 트위스트 페어 케이블
>>- 일반적으로 랜 케이블이라 부르며 UTP 케이블과 STP 케이블이 있음
>>- UTP 케이블
>>>- 구리 선 여덟 개를 두 개씩 꼬아 만든 네 쌍의 전선
>>>- 실드(shield)로 보호되어 있지 않음
>>>>- 금속 호일이나 금속의 매듭과 같은 것으로 노이즈를 막는 역할
>>>- 저렴하여 일반적으로 많이 사용됨
>>>- 분류|규격|속도
>>>   :--|--:|:--
>>>   Cat3|10 BASE-T|10Mbps
>>>   Cat5|100 BASE-TX|100Mbps
>>>   Cat5e|1000 BASE-T|1000Mbps
>>>   Cat6|1000 BASE-TX|1000Mbps
>>>   Cat6a|10GB BASE-T|10GMbps
>>>   Cat7|10GB BASE-T|10GMbps
>>- STP 케이블
>>>- 두 개씩 꼬아 만든 선을 실드로 보호한 케이블
>>>- 노이즈의 영향을 매우 적게 받지만 비싸서 보편적으로 사용하지 않음
>>- 랜 케이블의 양쪽 끝의 RJ-45 커넥터를 통해 컴퓨터의 랜 포트나 네트워크 기기에 연결됨
>- 다이렉트 케이블과 크로스 케이블
>>- 랜 케이블의 종류
>>- 다이렉트 케이블
>>>- 구리 선 여덟 개를 같은 순서로 커넥터에 연결한 케이블
>>>- 한쪽 컴퓨터에선 1,2번으로 송신하고 3번,6번으로 수신
>>>- 다른 한쪽 컴퓨터에선 1,2번으로 수신하고 3번,6번으로 송신
>>>- 컴퓨터와 스위치를 연결할 때 사용
>>- 크로스 케이블
>>>- 구리 선 여덟 개 중 한쪽 커넥터의 1번과 2번에 연결되는 구리 선은 다른 쪽 커넥터의 3번과 6번에 연결한 케이블
>>>- 양쪽 컴퓨터 모두 1,2번으로 수신하고 3,6번으로 송신
>>>- 컴퓨터 간에 직접 랜 케이블로 연결할 때 사용
>>- 1,2,3,6번 이외의 다른 선은 사용하지 않음

>### 리피터와 허브의 구조
>- 리피터
>>- 전기 신호를 정형(일그러진 전기 신호를 복원)하고 증폭하는 기능을 가진 네트워크 중계 장비
>>- 일대일 통신만 가능
>>- 최근엔 다른 네트워크 장비가 리피터 기능을 지원하여 사용하지않음
>- 허브
>>- 리피터와 똑같이 전기 신호를 정형하고 증폭하는 기능을 가진 네트워크 중계 장비
>>- 포트(실제 통신하는 통로)를 여러 개 가지고 있고 리피터 허브라고도 불림
>>- 리피터와 달리 포트를 여러 개 가지고 있어 컴퓨터 여러 대와 통신 가능
>>- 특정 포트로부터 데이터를 받으면 해당 포트를 제외한 나머지 모든 포트로 데이터를 전송하는 특징
>>>- 판단 없이 전기 신호를 모든 포트로 보내는 특성때문에 더미 허브라는 이름으로도 불림
>>>>- 대책으로 데이터 링크 계층의 스위치라는 네트워크 장비가 있음

## 데이터 링크 계층: 랜에서 데이터 전송하기

>### 데이터 링크 계층의 역할과 이더넷
>- 데이터 링크 계층
>>- 네트워크 기기 간에 데이터를 전송하고 물리주소를 결정
>>- 일반적으로 많이 사용되는 규칙이 이더넷(Ethernet)
>- 이더넷
>>- 허브와 같은 장비에 연결된 컴퓨터와 데이터를 주고받을 때 사용하는 기술 규격
>>- 허브에 여러 컴퓨터가 동시에 데이터를 전송해도 충돌이 일어나지 않는 구조
>>>- 충돌을 방지하기 위해 데이터를 보내는 시점을 늦춤
>>>- 시점을 늦추는 방법으로 CSMA/CD 방식 사용
>>>>- Carrier Sense Muliple Access with Collision Detection: 반송파 감지 다중 접속 및 충돌 탐지
>>>- 현재는 효율이 좋지 않아 CSMA/CD 사용하지 않고 스위치를 통해 충돌을 막음

<br>

[목차로 이동](#목차)  

>### MAC 주소의 구조
>- MAC 주소 (Media Access Control Address)
>>- 랜에 사용되는 네트워크 모델인 이더넷의 물리적인 주소로 컴퓨터 네트워크에서 각각의 기기를 구분하기 위해 사용하는 주소
>>>- 48비트로 구성되며 24비트(제조사번호) + 24비트(제조사가 붙인 일련번호)
>- 이더넷 헤더와 트레일러
>>- TCP/IP의 네트워크계층, OSI의 데이터 링크 계층에서 이더넷 헤더와 트레일러를 붙임
>>- 이더넷 헤더
>>>- 목적지 MAC 주소(6바이트) + 출발지 MAC 주소(6바이트) +  유형(2바이트) 로 구성
>>>>- 이더넷 유형: 이더넷으로 전송되는 상위 계층 프로토콜의 종류를 나타냄
>>>>- 유형 번호|프로토콜
>>>>   :--|:--
>>>>   0800|IPv4
>>>>   0806|ARP
>>>>   8035|RARP
>>>>   814C|SNMP over Ethernet
>>>>   86DD|IPv6
>>- 트레일러
>>>- FCS(Frame Check Sequence)라고도 하며 데이터 전송 도중 오류가 발생하는지 확인하는 용도로 사용
>>- 이더넷 헤더와 트레일러가 추가된 데이터를 프레임이라 함

>### 스위치의 구조
>- 스위치
>>- 랜을 구성할 때 사용하는 단말기 간 스위칭 기능이 있는 통신망 중계 장치로 호스트에서 특정한 다른 단말기로 패킷을 보낼 수 있는 기능이 있어 통신 효율이 향상됨
>>- 데이터 링크 계층에서 동작하고 레이어 2 스위치 또는 스위칭 허브라고 불리며 허브와 외형이 비슷함
>>- 스위치 내부에는 MAC 주소 테이블이 있음
>- MAC 주소 테이블
>>- MAC 주소 테이블은 스위치의 포트 번호와 해당 포트에 연결되어 있는 컴퓨터의 MAC 주소가 등록되는 데이터 베이스임
>>- MAC 주소 학습 기능
>>>- 컴퓨터에서 목적지 MAC 주소가 추가된 프레임이라는 데이터가 전송되면 MAC 주소 테이블을 확인하고 출발지 MAC 주소가 등록되어 있지 않으면 MAC 주소를 포트와 함께 등록함
>- 전송 과정
>>- 데이터 전송시 MAC 주소 테이블에 목적지 MAC 주소가 등록되어 있지 않다면 송신 포트 이외의 모든 포트로 데이터가 전송되는데 이를 플러딩(flooding)이라 함
>>>- 최초 전송 과정에서 MAC 주소 테이블에 MAC 주소를 저장함
>>- MAC 주소 테이블에 등록되어 있는 주소라면 해당 포트에만 데이터 전성됨
>>>- MAC 주소를 기준으로 목적지를 선택하는 것을 MAC 주소 필터링이라 함

## 네트워크 계층: 목적지에 데이터 전달하기

>### 네트워크 계층
>- 네트워크 계층의 역할
>>- 다른 네트워크와 통신하기 위한 경로 설정을 위해 라우터를 통한 라우팅을 하며 패킷 전송을 담당
>- 라우터
>>- 데이터의 목적지가 정해지면 해당 목적지까지 어떤 경로로 가는것이 좋은지 알려주는 기능을 함
>>- 네트워크 계층을 통해 다른 네트워크로 데이터를 전송하려면 라우터가 필요
>>- 목적지가 어떤 네트워크의 어떤 컴퓨터인지 식별할 수 있는 IP 주소를 사용함
>>- 목적지 IP 주소까지 어떤 경로로 데이터를 보낼지 결정하는 것을 라우팅이라고 함
>>>- 라우팅을 하려면 라우터가 필요하며 라우터에는 라우팅 테이블이 있어 경로 정보를 등록하고 관리함
>- IP (Internet Protocol)
>>- 인터넷에 있는 한 컴퓨터에서 다른 컴퓨터로 데이터를 보내는 데 사용되는 네트워크 계층 프로토콜
>>- TCP/IP의 IP와 같음
>>- 네트워크 계층에서의 캡슐화는 IP헤더를 붙임
>>>- 버전, 헤더길이, 서비스유형, 전체패킷길이, ID, 조각상태, 조각의위치, TTL, 프로토콜, 헤더체크섬, 출발지IP주소, 목적지IP주소 순서로 IP 헤더 정보가 구성되어 있음
>>>- IP 프로토콜로 캡슐화 된 것을 IP 패킷이라 함

>### IP 주소의 구조
>- IP주소
>>- 네트워크에서 장치들이 서로를 인식하고 통신하기 위해 사용하는 주소로 인터넷 서비스 제공자(ISP)가 제공
>>- IPv4 와 IPv6
>>>- IPv4: 32비트로 이루어진 주소로 약 43억 개를 만들 수 있어 부족
>>>- IPv6: 128비트로 이루어진 주소로 약 230간 개를 만들 수 있어 매우 충분
>>- 공인 IP 주소와 사설 IP 주소
>>>- 공인 IP 주소: 인터넷에 직접 연결되는 컴퓨터나 라우터에 할당되며 인터넷 서비스 제공자(ISP)가 제공
>>>- 사설 IP 주소: 회사나 가정의 랜에 있는 컴퓨터에 할당되며 랜의 네트워크 관리자가 할당하거나 라우터의 DHCP 기능으로 자동 할당되어 제공
>>- 일반적으로 8비트씩 10진수로 계산하여 끊어 읽음 ex) 127.0.0.1
>>>- IP주소는 네트워크 ID와 호스트 ID로 나뉨

## 전송계층: 신뢰할 수 있는 데이터 전송하기

>### 전송계층의 역할
>- 신뢰할 수 있는 데이터를 순차적으로 전달하는 역할을 하므로 상위 계층들이 데이터 전달의 유효성이나 효율성을 신경 쓰지 않도록 함
>- 데이터가 중복되거나 누락되지 않고 오류 없이 순서에 맞게 전송되도록 관리함
>- 전송계층의 두 가지 기능
>>- 목적지에 신뢰할 수 있는 데이터를 전달하기 위해 오류를 점검하는 기능
>>>- 오류가 발생하면 데이터를 재전송하도록 요청
>>- 전송된 데이터의 목적지가 어떤 애플리케이션인지 식별하는 기능
>- 연결형 통신과 비연결형 통신
>>- 연결형 통신
>>>- 데이터를 교환하기 전에 연결을 맺고 데이터를 교환하는 동안 계속 연결을 관리하는 프로토콜의 한 형태
>>>- 데이터를 목적지에 문제없이 전달하는 것
>>>- TCP 사용
>>- 비연결형 통신
>>>- 연결에 대한 초기화 과정이 없는 통신
>>>- 데이터를 빠르고 효율적으로 전달하는 것
>>>- UDP 사용

<br>

[목차로 이동](#목차)

>### TCP의 구조
>- TCP란
>>- 신뢰성과 정확성을 우선으로 하는 연결형 통신 프로토콜
>- 세그먼트와 TCP헤더
>>- TCP로 전송할 때 붙이는 헤더를 TCP헤더라 함
>>- TCP 헤더가 붙은 데이터를 세그먼트라 함
>>- TCP 헤더의 구조
>>>- 출발지포트번호+목적지포트번호+일련번호+확인응답번호+헤더길이+예약영역+코드비트+윈도우크기+체크섬+긴급포인터+옵션
>- 연결형 통신
>>- 연결형 통신에서는 데이터를 전송하기 전에 연결(connection)이라는 가상의 독점 통신로를 확보해야함
>>- TCP 헤더 중 코드 비트를 통해 연결의 제어 정보가 기록됨
>>>- 코드비트의 구조: URG+ACK+PSH+RST+SYN+FIN
>>- 연결을 확립하려면 코드비트의 SYN와 ACK을 사용
>>>- SYN: 연결 요청
>>>- ACK: 확인 응답
>- 3-way 핸드셰이크
>>- 신뢰할 수 있는 연결을 하기 위해 데이터를 전송하기 전 세 번의 패킷교환이 진행됨
>>>1. 컴퓨터1: 연결 확립 요청(SYN)
>>>2. 컴퓨터2: 연결 확립 응답(ACK) + 연결 확립 요청(SYN)
>>>3. 컴퓨터1: 연결 확립 응답(ACK)
>>- 데이터 전송이 끝난 후 연결을 끊기 위한 요청도 필요
>>>1. 컴퓨터1: 연결 종료 요청(FIN)
>>>2. 컴퓨터2. 연결 종료 응답(ACK)
>>>3. 컴퓨터2. 연결 종료 요청(FIN)
>>>4. 컴퓨터1. 연결 종료 응답(ACK)

>### 일련번호와 확인 응답 번호의 구조
>- 핸드셰이크가 끝나고 나면 일련번호와 확인 응답 번호를 사용함
>>- 일련번호: 송신 측에서 수신 측에 현재 데이터가 몇 번째 데이터인지 알려주는 역할
>>- 확인 응답 번호: 수신 측이 몇 번째 데이터를 수신했는지 송신 측에 알려주는 역할
>>- 200바이트씩 전송될 때 송수신과정
>>>1. 컴퓨터1: 일련번호 3001, 확인응답번호 4001 전송
>>>1. 컴퓨터2: 일련번호 4001, 확인응답번호 3201 전송
>>>1. 컴퓨터1: 일련번호 3201, 확인응답번호 4001 전송
>>>1. 컴퓨터2: 일련번호 4001, 확인응답번호 3401 전송
>- 재전송 제어
>>- 일련번호와 확인 응답 번호를 사용하여 데이터가 손성되거나 유실된 경우 데이터를 재전송
>- 윈도우크기
>>- 세그먼트 하나를 보낼 때마다 확인 응답을 반환하는 방식보다 연속해서 보낸 후 확인 응답을 반환하는것이 효율적
>>>- 버퍼를 통해 세그먼트를 일시적으로 보관
>>- 버퍼의 오버플로가 발생하지 않도록 TCP 헤더의 윈도우 크기 값을 활용하여 전송
>>- 핸드셰이크 과정에서 윈도우 크기 값을 결정함

>### 포트번호의 구조
>- 포트번호
>>- 컴퓨터가 데이터 통신을 할 때 통신하고자 하는 네트워크 서비스나 특정 프로세스를 식별하는 논리 단위
>>- 전송된 데이터의 목적지가 어떤 애플리케이션인지 구분하는 역할
>>-  포트번호의 범위: 0~65535
>>>- 0~1023: 잘 알려진 포트로 주요 프로토콜이 사용하도록 예약
>>>- 1024: 예약되어있지만 사용하지않음
>>>- 1025이상: 랜덤 포트로써 클라이언트 측의 송신 포트로 사용
>>>- 애플리케이션|포트 번호
>>>   :--|:--
>>>   SSH|22
>>>   SMTP|25
>>>   DNS|53
>>>   HTTP|80
>>>   POP3|110
>>>   HTTPS|443
